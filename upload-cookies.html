2. Three ways to get the file to the VPS

This page uploads directly to the running Vercel API. If that doesn't work for any reason, here are three other paths — pick whichever your laptop allows.

Method A — Direct upload via this page (easiest, what the form does)

Just pick the file above and click Upload. The browser sends it to a Vercel function which writes it to /root/.agent-reach/ on the VPS.

What it does: file is uploaded over HTTPS, validated, saved with a timestamp suffix so duplicates don't collide, and triggers a Telegram notification to Joe.

Method B — scp from your laptop terminal (most reliable)

If you have SSH access to the VPS, this is the fastest path. On your laptop:

scp youtube.com_cookies.txt root@72.61.141.125:/root/.agent-reach/

(Replace youtube.com_cookies.txt with your actual filename.)

You'll be prompted for the root password. If you don't have it / SSH times out, use Method C below.

Method C — Telegram (works from any device, no setup)

Send the file via Telegram direct message to @JoeSaasAgencyBot with the caption:

/upload-cookies youtube

Hermes will pick it up, save it to /root/.agent-reach/youtube.com_cookies.txt, and run agent-reach configure youtube-cookies automatically.

Note: this is a planned feature. If it's not live yet, the bot will tell you to use Methods A or B.

After the file lands — what happens automatically

Once a cookies file is saved to /root/.agent-reach/, the next agent run will:

  1. Detect the new file via the watcher
  2. Run agent-reach configure <service>-cookies <path>
  3. Run agent-reach doctor --verify-cookies to confirm the cookies work
  4. Send you a Telegram message: "✅ <service> cookies loaded. You can now <what it unlocks>."
  5. On failure: "❌ cookies loaded but <service> rejected them. Re-export from a fresh browser session."

3. How to export cookies.txt from your browser

Chrome (laptop)
  1. Install the free extension "Get cookies.txt LOCALLY" (by Rahul Baruri) from the Chrome Web Store
  2. Open the target site (e.g. youtube.com) — make sure you're logged in
  3. Click the extension icon → "Export cookies for this tab" (or "Export" → save as .txt)
  4. The downloaded file will be named after the site (e.g. youtube.com_cookies.txt) — that's what you upload
Firefox (laptop)
  1. Install "cookies.txt" by danylchuk (or similar)
  2. Same as Chrome: open the site, click the extension, export
Safari (Mac)
  1. Use the built-in Develop menu: Develop → Show Web Inspector → Storage → Cookies → right-click → Export
  2. Or install the "Cookie-Editor" extension
Edge
  1. Use the same Chrome extension — Edge supports Chrome Web Store extensions natively

What to do if the file won't upload

  • File too large (> 1MB): cookies.txt files are usually < 50KB. If yours is huge, you exported the wrong thing — try again from a fresh browser session.
  • "Rejected" error: cookies contain only browser-internal cookies (no session cookies). Log out, log in again, re-export.
  • Page won't load on phone: try the form on a laptop browser instead. Or use Method B (scp).
  • Network issue on Vercel: use Method B (scp) as a fallback — it bypasses Vercel entirely.
